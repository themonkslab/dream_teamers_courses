# Errores y excepciones

Hasta ahora venimos viéndolos pero nunca hablamos detenidamente de ellos y qué diferencias podemos encontrar entre un tipo y el otro. Vamos a por ello!

## _Errors_

Los _errors_ se refieren a __situaciones que generalmente no se pueden manejar durante la ejecución del programa y que causan la terminación instantánea del mismo__. Ejemplos? Errores de compilación, cuando está tratando de ejecutar alguna función dentro de una librería y no la encuentra o errores de tiempo de ejecución! Veamos algunos de éstos últimos, con ejemplos:

- _RangeError_: Se produce cuando se intenta acceder a un índice fuera de los límites de una lista o colección.

```dart
final List<int> numbers = [1, 2, 3];
print(numbers[3]); // Lanza un RangeError porque el índice 3 está fuera de los límites de la lista
```

- _TypeError_: Se produce cuando se intenta realizar una operación en un tipo de dato incompatible.

```dart
final int number = 42;
print(number.toUpperCase()); // Lanza un TypeError porque los enteros no tienen un método 'toUpperCase'
```

- _NoSuchMethodError_: Se produce cuando se intenta llamar a un método que no existe en un objeto.

```dart
final String name = 'John';
print(name.notAMethod()); // Lanza un NoSuchMethodError porque el método 'notAMethod' no existe en String
```

- _FormatException_: Se produce cuando se intenta convertir un valor de un tipo a otro, pero el valor no se puede convertir correctamente.

```dart
final String value = 'not a number';
print(int.parse(value)); // Lanza un FormatException porque 'not a number' no es un número válido
```

Estos son solo algunos ejemplos de errores en tiempo de ejecución que pueden ocurrir en Dart.

## _Exceptions_

Por otro lado, las excepciones son __eventos que ocurren durante la ejecución del programa que pueden ser manejados por el código__. Se modelan con la clase
_Exception_ y sus subclases y se pueden lanzar explícitamente mediante la palabra clave throw, o pueden ser lanzadas automáticamente por el
sistema cuando ocurren ciertos errores de tiempo de ejecución, como un índice fuera de rango o una llamada a un método en un objeto nulo.

En resumen, los errores son problemas fatales que no se pueden manejar, mientras que las excepciones son eventos que pueden ser manejados y resueltos en tiempo de ejecución.
