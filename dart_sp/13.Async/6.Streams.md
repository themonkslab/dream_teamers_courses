## Stream generators

__Un _stream generator_ es una funci√≥n que produce un flujo de datos (_stream_)
en el que se pueden emitir valores de forma asincr√≥nica a medida que se van
produciendo__. A diferencia de una funci√≥n normal, que retorna un valor y
finaliza su ejecuci√≥n, un _stream generator_ puede seguir emitiendo valores
indefinidamente, siempre y cuando no se detenga expl√≠citamente su ejecuci√≥n.

La sintaxis es similar a la de una funci√≥n as√≠ncrona, pero en lugar de utilizar
el tipo de retorno `Future` se utiliza el tipo `Stream`. Adem√°s, se utiliza la
palabra clave `async*` en lugar de `async`y se utiliza la sentencia `yield` para
emitir valores en el _stream_ en lugar de la sentencia `return` para retornar un
valor.

```dart

  // Ahora vamos a crear un Stream mediante un generador as√≠ncrono.
  // Para ello, vamos a crear una funci√≥n que retorne un Stream<int> mediante el
  // operador async* y que reciba un par√°metro to de tipo int.
  Stream<int> countStream(int to) async* {
    // Dentro de la funci√≥n, vamos a iterar desde 1 hasta el valor de to
    for (int i = 1; i <= to; i++) {
      // Por cada iteraci√≥n, vamos a esperar 1 segundo
      await Future.delayed(Duration(seconds: 1));
      // Y vamos a emitir el valor de i en el Stream
      yield i;
    }
  }

  final count = countStream(5);

  count.listen((event) async {
    print('Stream new value: $event');
  });
}
```

Si prueban el ejemplo, van a ver ahora s√≠, como los valores se emiten con un
intervalo de un segundo entre cada uno. Pueden remover el `yield i` para
observar que entonces nada sucede, ya que lo que hace es emitir el valor que
luego vamos a escuchar y va a convertirse en evento.
